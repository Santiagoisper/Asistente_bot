**Rama:** `main`  
**Estado:** implementación completa en código; **migración DB pendiente de aplicar**

## Resumen en 30 segundos

Se implementaron tres frentes del plan acordado:

1. **Chat SNOMED híbrido** — preguntas de codificación ya no caen en "Evidencia insuficiente" cuando el diccionario tiene el código; la UI muestra protocolo vs sugerencia externa.
2. **Tipografía** — Inter vía `next/font`, texto de chat más legible (15–16px).
3. **Librería de protocolos** — bulk import, catálogo org (`/library`), columna `terminology_annotations` y tabla `ingestion_jobs`.

## Qué probar manualmente

### 1. SNOMED en el chat

1. Abrí un estudio con protocolo indexado (ej. mock metabólico).
2. Preguntá: *"diabetes tipo 1, no tenés referencia en snomed-ct?"* o *"¿Lo podés asociar a un código de SNOMED-CT?"*
3. **Esperado:**
   - No badge rojo "Evidencia insuficiente" (si hay match en diccionario).
   - Tarjeta teal **"Codificación sugerida"** con badge **"Sugerencia externa"**.
   - Código `46635009` para diabetes tipo 1.
   - Disclaimer: el código no proviene del protocolo.

### 2. Tipografía

- Texto del chat en ~15px, fuente Inter cargada.
- Badges y citas menos agresivos (menos uppercase extremo).

### 3. Bulk import

- UI: `/studies/import` — drag & drop de varios PDFs.
- API: `POST /api/studies/bulk-import` (multipart, campo `files`).
- Polling: `GET /api/studies/bulk-import/{batchId}`.
- CLI: `node scripts/bulk-import-protocols.mjs --dir ./mis-protocolos --user-id <CLERK_USER_ID>`

### 4. Librería org

- `/library` — listado filtrable con conteos de spec, docs indexados, chips SNOMED.
- Links a chat, spec y documentos por protocolo.

### 5. Terminología al aprobar spec

- Aprobá un spec en `/studies/[id]/spec`.
- La columna `terminology_annotations` se llena con códigos SNOMED/LOINC del diccionario.
- La librería los muestra sin recalcular (fallback legacy: jsonb embebido en `spec` pre-migración).

## Decisiones de diseño (para no re-debatir)

| Decisión | Por qué |
|----------|---------|
| SNOMED sugerido ≠ contenido del protocolo | Regla CLAUDE.md + ADR-004; disclaimer visible; confianza máx `medium`. |
| Diccionario local, no API live en stream | Latencia y español no soportado en Ontoserver. |
| `ingestion_jobs` + columna dedicada | Versión formal vs jsonb embebido / polling por estudio. |
| Bulk = 1 PDF = 1 estudio | Aislamiento tenant; no RAG cross-study. |
| Reintentos ingestión = 3 | `max_attempts` default en schema. |
